## 🔄 Data Flow Diagram

```mermaid
sequenceDiagram
    participant User
    participant CLI as ArchDoc CLI
    participant Agents as Analysis Agents
    participant Cache as JSON Cache
    participant Export as Export Service
    participant Dashboard as React Dashboard
    participant Host as Hosting Platform
    participant Browser as User's Browser

    User->>CLI: archdoc analyze ./project
    CLI->>Agents: Run analysis

    loop For each agent
        Agents->>Agents: Analyze codebase
        Agents->>Cache: Write JSON results
    end

    Agents-->>CLI: Analysis complete
    CLI-->>User: ✓ Results saved to .archdoc/cache/

    User->>CLI: archdoc export --format html-interactive
    CLI->>Export: Generate dashboard
    Export->>Cache: Read JSON files
    Cache-->>Export: Return data
    Export->>Dashboard: Generate React site
    Dashboard->>Dashboard: Build static files
    Dashboard-->>CLI: Build complete
    CLI-->>User: ✓ Dashboard at ./archdoc-dashboard/

    User->>CLI: archdoc deploy --platform github-pages
    CLI->>Host: Upload static files
    Host-->>CLI: Deployment successful
    CLI-->>User: ✓ Live at https://username.github.io/docs

    User->>Browser: Open dashboard URL
    Browser->>Host: Request index.html
    Host-->>Browser: Return HTML/CSS/JS
    Browser->>Browser: Load React app
    Browser->>Browser: Read JSON from archdoc-data/
    Browser-->>User: Display interactive dashboard
```

## 📊 Data Schema: JSON Structure

```mermaid
erDiagram
    ARCHITECTURE-ANALYSIS ||--o{ COMPONENT : contains
    COMPONENT ||--o{ DEPENDENCY : has

    NODE ||--o{ EDGE : connects

    PENTEST-ANALYSIS ||--o{ ENDPOINT : maps

    VULNERABILITY ||--o{ REMEDIATION : suggests

    THREAT-MODEL ||--o{ STRIDE-THREAT : contains
    STRIDE-THREAT ||--o{ ATTACK-PATH : defines
    ATTACK-PATH ||--o{ MITIGATION : requires

    USER-FLOWS ||--o{ FLOW : contains
    FLOW ||--o{ STEP : has
    STEP ||--o{ FILE : references

    COMPONENT {
        string name
        string type
        string layer
        int riskScore
        array files
    }

    ENDPOINT {
        string path
        string method
        string auth
        int riskLevel
        array vulnerabilities
    }

    VULNERABILITY {
        string type
        string severity
        float cvssScore
        string owaspCategory
        string description
    }

    FLOW {
        string name
        int stepCount
        string riskLevel
        array steps
    }
```

## 📈 Update Workflow

```mermaid
stateDiagram-v2
    [*] --> CodeChange: Developer commits code

    CodeChange --> Analysis: CI/CD triggers analysis
    Analysis --> CacheUpdate: Update JSON cache
    CacheUpdate --> DashboardBuild: Generate new dashboard
    DashboardBuild --> Deploy: Deploy to hosting
    Deploy --> Live: Dashboard updated

    Live --> [*]: Users see latest docs

    note right of Analysis
        archdoc analyze .
        Runs all agents
    end note

    note right of DashboardBuild
        archdoc export
        Builds React site
    end note

    note right of Deploy
        archdoc deploy
        Pushes to hosting
    end note
```

## 🎯 Summary

These schemas show:

1. **System Architecture**: How CLI, agents, cache, and dashboard connect
2. **Data Flow**: Step-by-step process from analysis to deployment
3. **File Structure**: Where files are stored and how they're organized
4. **Deployment**: Multiple hosting options and workflows
5. **User Interaction**: How users navigate the dashboard
6. **Security Analysis**: How pentest features work
7. **CI/CD Integration**: Automated updates via GitHub Actions
8. **Data Schema**: JSON structure relationships
9. **Component Architecture**: React component hierarchy
10. **Deployment Architecture**: CDN and global distribution
11. **Update Workflow**: How docs stay current

All diagrams are in **Mermaid format** and can be rendered in GitHub, VS Code, or any Markdown viewer that supports Mermaid!
